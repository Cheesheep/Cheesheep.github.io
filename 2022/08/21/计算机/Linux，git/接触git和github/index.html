接触git和GitHub

接触git和Github

  1. 首先是通过csdn上的博客查阅资料得到相关内容，大概了解了git大概是个什么东西，然后知道了要怎么连接上github，不过目前还有许多不清楚的
  2. 然后是我在ubuntu里面下载了一个本地git，现在正在尝试与github连接上（使用SSH密钥通讯方式）

Linux下的git

1.创建ssh公钥，连接上github（也可以直接用http）但是会比较麻烦

首先账号是LINUXSHEEP-git好像

然后密钥默认存储路径为home/flasheep/.ssh

设置密钥

但是.ssh是有权限的，只能通过命令行找到并且打开

.ssh目录下由两个文件，一个是id_rsa,一个是id_rsa.pub，后者为公钥，用于放到github当中作为key，前者为密钥，不能透露

以下两篇博客给了我很大的帮助

真小白入门之Github_nmjuzi的博客-CSDN博客_github入门

💡 这里新增一下，连接github公钥大概率接不上，会出现 ****git报错ssh: connect to host [github.com](http://github.com) port 22: Connection timed out****

如何解决呢

找到原因后我们进入.ssh的目录，使用命令“touch config”创建一个配置文件(不需要后缀，可以先用txt写入再改回去)，并写入你github的配置信息。（xxxxx@xx.com是你github的注册邮箱）

现在再使用ssh git@github.com查看与github的连接状态，可能出现错误Bad owner or permissions on

这是因为你创建的配置文件config的权限不够。所以我们需要修改其权限。使用下面命令。

sudo chmod 600 config
1
现在我们再尝试查看连接状态发现已经成功连接。

接下来就可以成功的push你的代码到github了。

2.将本地库上传到远程库（即github）

创建一个git库并连接GitHub远程库（大致步骤）（默认已经生成了该机器的ssh公钥以及和GitHub的setting中关联上了）

1•首先要下载一个git

windows与Linux稍有不同，Linux可以直接在终端下载，要使用git，要使用命令时直接用，Windows好像还要下载一个程序，然后进入git bash

创建ssh公钥并与GitHub关联

2·在GitHub中新建一个repository，假设命名为learngit

可以是空项目，也可以是里面放什么都行

3•建立git本地库

随便创建一个新的文件夹，在该目录下初始化仓库

如果未给本机器的git命名，则可以命名，记得邮箱要和GitHub的相同

初始化之后该文件夹会有一个隐藏目录.git（通过 ls -ah）可以查看到

4·本地库与GitHub远程库连接

origin是对本地库来说连接了的远程库的名称

连接成功后，将远程仓库（可以是一个空仓库）克隆到本地

5·将本地的文件上传到缓存区与git本地库

以上只是提交到缓存区，接下来提交到版本库也就是git本地库

以上步骤完成后会建立了一条master主分支，可以对该分支做进一步分枝branch

6·将主分支master上传至远程库就成功啦

如果远程库命名不是origin记得要改

7.一次性上传所有文件的方法

  1. git add -A .

    git add xx命令可以将xx文件添加到暂存区，如果有很多改动可以通过 git add -A .来一次添加所有改变的文件。注意 -A 选项后面还有一个句点。 git add -A表示添加所有内容， git add . 表示添加新文件和编辑过的文件不包括删除的文件; git add -u 表示添加编辑或者删除的文件，不包括新添加的文件

  2. git commit -m “提交注释”

  3. git push origin 分支名称，一般使用：git push origin master

VScode + git

  • 至于vscode和git的安装就不在这里展开了，网上一搜一大把
  • 要是等你熟悉了git之后，可以尝试用vscode来可视化git的操作管理，当然了没熟悉之前也可以用vscode来接触git，只不过有些地方可能用代码来讲述会更加容易理解一点
  • vscode的好处就是可以将git的操作给完全可视化，你既可以在控制台来完成git的操作，也可以通过vscode来完成相应的操作，两者本质上是一样的，所以可以放心用
  • vscode这边可以实时展现出哪些地方被更改了，而且还可以列出之前的版本来对照你的改动
  • 可以说是很方便了，不用每次调用git log来查看
